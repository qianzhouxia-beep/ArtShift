## Key Reasoning

### 问题背景
- ArtShift 是一个 Vite + React 前端项目（Landing Page）
- 之前在 Zeabur 部署失败，报错：`copy root directory "frontend": no such file or directory`
- 根因：旧服务缓存了错误的 Root Directory 配置

### 解决方案
1. **删除旧服务** — 清除所有缓存的错误配置
2. **重新创建服务** — 从 GitHub 仓库 `qianzhouxia-beep/ArtShift` 重新添加
3. **使用自动检测** — 让 Zeabur 自动识别项目类型（nodejs + vite）
4. **关键修复**：
   - 添加 `serve` 包和 `start` 脚本到 package.json（解决 502）
   - 设置 `vite.config.ts` 的 `base: './'`
   - 替换 Logo 为图标版（A + 像素渐变，透明背景）

### 📌 关键经验
1. Zeabur 缓存错误配置时，删除重建比修改配置更可靠
2. Vite 项目 Output Directory 必须是 `dist`
3. 静态站点需要 start script（如 `serve dist -l $PORT`）
4. 新版 Zeabur 的"构建计划预览"会自动检测配置，通常不需要手动设置 Root Directory
