## Favicon & Icons Diagnosis (00:33-00:39 GMT+8)

### Issue Reported
User reported `favicon.svg` and `icons.svg` appear as "white boards" (blank) when opened

### Diagnosis Results

**1. favicon.svg**
- Contains: `<image href="/favicon.png" width="48" height="48"/>`
- Problem: Absolute path `/favicon.png` doesn't resolve when opening SVG file directly
- Actual usage: **Works fine** in browser (served from website root)
- index.html already correctly configured with both PNG and SVG favicon links

**2. icons.svg** ✅ **NORMAL BEHAVIOR**
- This is an **SVG sprite sheet** containing 6 symbol definitions:
  - `bluesky-icon`
  - `discord-icon`
  - `documentation-icon`
  - `github-icon`
  - `social-icon`
  - `x-icon`
- Blank when opened directly - **this is expected!**
- Usage: `<use href="icons.svg#github-icon" />` (embedded in other SVGs)

### Resolution
- **No code changes needed**
- Current setup in `index.html` is correct:

- SVG sprite sheets ALWAYS appear blank when opened directly - this is normal
- Absolute paths in SVGs only work when served by a web server, not file:// protocol

## Favicon Not Showing in Browser Tab (01:00-01:10 GMT+8)

### Issue
User sent screenshot showing browser tab with **blank favicon** — ArtShift site title visible but no icon on left side of tab.

### Investigation
- Checked live site: `favicon.png` returns 200 OK, valid PNG 512×512
- The actual PNG logo image is beautiful: dark blue "A" letter with purple-to-blue pixel art gradient
- File itself works fine — problem was in how browsers load it

### Root Cause
Old `favicon.svg` was a **shell/wrapper** that only contained:
```svg
<svg><image href="/favicon.png" width="48" height="48"/></svg>
```
Some browsers prioritize SVG favicon over PNG, but this shell SVG couldn't resolve the external path properly in all contexts, resulting in blank display.

### Fix Applied (Commit `98b0f6c`)

**1. Rewrote `public/favicon.svg` as self-contained inline SVG**
- Hand-drawn SVG matching the brand logo: stylized "A" + pixel art gradient blocks
- Uses purple (#8B5CF6) to blue (#3B82F6) linearGradient
- White rounded rect background, matches ArtShift brand identity
- No external dependencies — fully self-contained

**2. Updated `index.html` favicon links**
- Added `sizes="512x512"` attribute to PNG link

2. `rgba(255,255,255,0.02)` fade → trails **accumulate** → messy/chaotic on white sections
3. `opacity: 1.0` → too dark on white background sections

### Fix Applied (commit `latest`)
| Issue | Fix |
|-------|-----|
| zIndex 0 → hidden behind hero | `zIndex: 1` (above content, `pointerEvents: none`) |
| Trail accumulation | Replaced fade with `clearRect(0,0,w,h)` — no trails at all |
| Opacity 1.0 too dark | `opacity: 0.12 ~ 0.24` (subtle overlay feel) |
| Size 10-30px too big | `6-20px` (smaller, more delicate) |
| Speed 0.5 too fast | `0.3` (slower, more elegant) |
| Repel too strong | force multiplier `3` → `2` |
| Count 40 | `35` (fewer = less cluttered) |

**Result:** Particles now look like "light floating gauze" — visible on hero bg, not overwhelming on white sections.

## Key Lesson: Full-Screen Canvas Particle Design

When using full-screen canvas particles on a multi-section page:

1. **zIndex matters** — `pointer-events: none` + `zIndex: 1` = visible but non-blocking
2. **Trail effects on white backgrounds** = visual noise, not ambiance
3. **No-trail (clearRect) + low opacity** = works on ALL background types
4. **Particle size** should be 6-20px for full-screen; 10-30px is too chunky
